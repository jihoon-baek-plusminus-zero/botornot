# BotOrNot

웹 기반 프로젝트로 Supabase, GitHub, Vercel을 연동한 애플리케이션입니다.

## 기술 스택

- **Frontend**: Next.js 14, TypeScript, Tailwind CSS
- **Backend**: Supabase
- **Deployment**: Vercel
- **Version Control**: GitHub

## 환경 설정

### 필수 환경 변수

프로젝트 루트에 `.env.local` 파일을 생성하고 다음 환경 변수를 설정하세요:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
OPENAI_API_KEY=your_openai_api_key
```

## 설치 및 실행

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npm run dev

# 프로덕션 빌드
npm run build

# 프로덕션 서버 실행
npm start
```

## 배포

이 프로젝트는 Vercel을 통해 자동 배포됩니다:

1. GitHub 저장소에 코드를 푸시
2. Vercel에서 GitHub 저장소 연결
3. 환경 변수 설정
4. 자동 배포 완료

## AI vs Human 게임 웹앱: 제품 요구사항 정의서 (PRD) - v1.1 (상세화 버전)

### **1. 개요 (Overview)**

**1.1. 제품명**
AI vs Human (가제)

**1.2. 제품 설명**
'AI vs Human'은 사용자가 대화 상대가 인공지능(AI)인지 사람인지 추리하는 **실시간 익명 웹 기반 채팅 게임**입니다. 모든 게임 세션은 독립적이며, 게임 종료 시 모든 대화 기록과 데이터는 즉시 파기되어 개인정보 보호와 가벼운 플레이 경험을 보장합니다.

**1.3. 목표 (Goals)**
* **핵심 재미 제공:** 플레이어에게 대화 상대의 정체를 추리하는 과정에서 오는 긴장감과 재미를 선사합니다.
* **직관적인 사용자 경험:** 모바일 메신저와 유사한 친숙한 UI를 통해 누구나 쉽게 게임을 배우고 즐길 수 있도록 합니다.
* **높은 재플레이성:** 짧은 플레이 타임, 데이터가 남지 않는 가벼움, 매번 달라지는 AI 페르소나를 통해 반복적인 플레이를 유도합니다.

### **2. 사용자 페르소나 (User Personas)**

* **캐주얼 게이머 (20대, 대학생):** 친구들과 가볍게 즐길 수 있는 독특하고 새로운 컨셉의 온라인 게임을 찾고 있습니다. 복잡한 규칙보다는 직관적이고 빠르게 한 판을 즐길 수 있는 게임을 선호합니다.
* **AI/기술 관심가 (30대, 직장인):** 최신 AI 기술, 특히 자연어 처리(LLM)에 관심이 많습니다. AI가 얼마나 인간과 유사하게 대화할 수 있는지 직접 체험하고 테스트해보는 것에 흥미를 느낍니다.

### **3. 기능 요구사항 (Functional Requirements)**

#### **3.1. 메인 화면 (대기 화면)**
* **FR-101:** 화면 중앙에 게임 제목 또는 로고가 표시되어야 합니다.
* **FR-102:** "1:1 게임 시작" 버튼이 존재해야 합니다.
* **FR-103:** "1:N 게임 시작" 버튼이 존재해야 합니다.

---

#### **3.2. 게임 매치메이킹**
* **FR-201 (1:1 게임 - 가상 대기열 시스템):**
    * **FR-201-1:** 플레이어가 "1:1 게임 시작"을 누르면 가상 대기열에 입장합니다.
    * **FR-201-2:** 시스템은 대기열 순서에 따라 **[사람 vs 사람]** 매치와 **[사람 vs AI]** 매치를 번갈아 생성하여 매칭합니다.
        * (예: 1, 2번 입장 → 사람 매치 / 3번 입장 → AI 매치 / 4, 5번 입장 → 사람 매치 / 6번 입장 → AI 매치)
    * **FR-201-3:** 만약 플레이어가 대기열에서 **1분 이상** 대기할 경우, 시스템은 **1분 ~ 1분 30초 사이의 랜덤한 시점**에 해당 플레이어를 AI와 강제 매칭시킵니다.

* **FR-202 (1:N 게임 - 최소 인원 기반 시스템):**
    * **FR-202-1:** 플레이어가 "1:N 게임 시작"을 누르면 1:N 게임 대기열에 입장합니다.
    * **FR-202-2:** 대기열에 **사람 플레이어가 3명** 모이면, 시스템은 즉시 게임방을 생성합니다.
    * **FR-202-3:** 생성된 게임방에 **AI 플레이어 2명**을 추가하여 총 5명의 플레이어를 구성합니다.
    * **FR-202-4:** 5명의 플레이어에게 "Player A"부터 "Player E"까지의 라벨을 무작위로 배정합니다.
    * **FR-202-5:** 3명이 모일 때까지의 대기 시간 제한은 없습니다 (무한 대기).

---

#### **3.3. 게임 화면 및 플레이어**
* **FR-301:** UI는 모바일 메신저 형태를 유지합니다.
* **FR-302:** 화면 좌측 상단에 실시간 게임 타이머(1:1-2분, 1:N-5분)가 표시됩니다.
* **FR-303:** 화면 상단 중앙에 **미리 정의된 100개의 리스트**에서 무작위로 선택된 '대화 주제'가 표시됩니다.
* **FR-304:** 주제 옆에 "n/N Selected Vote" 상태가 실시간으로 표시됩니다.
* **FR-305:** "Vote" 버튼이 항상 활성화되어 있습니다.
* **FR-306:** 모든 플레이어는 **완전 익명**이며, "Player A" ~ "Player E"와 고유 색상으로만 구분됩니다. 로그인, 닉네임 기능은 없습니다.

---

#### **3.4. 채팅 및 게임 진행**
* **FR-401:** 채팅은 정해진 순서(A→B→...→A)에 따라 턴제로 진행됩니다.
* **FR-402:** 자신의 턴에 10초 내 미입력 시 "Skipped"가 자동으로 전송됩니다.

---

#### **3.5. 게임 종료 및 투표**
* **FR-501:** 모든 플레이어 'Vote' 선택 또는 시간 종료 시 게임이 끝나고 10초 투표가 시작됩니다.
* **FR-502 (1:1 투표):** "AI" 또는 "Human" 중 하나를 선택합니다.
* **FR-503 (1:N 투표):** 자신을 제외한 4명의 플레이어 중 2명을 선택합니다.

---

#### **3.6. 결과 화면**
* **FR-601:** 투표 종료 후 즉시 결과가 표시됩니다.
    * **1:1 결과:** 상대 정체, 나의 추측 결과, 상대의 나에 대한 추측 결과를 모두 표시합니다.
    * **1:N 결과:** 플레이어별 득표수와 실제 AI였던 2명을 명확히 공개합니다.
* **FR-602:** "메인 화면으로 돌아가기" 버튼이 존재하며, 클릭 시 메인 화면으로 이동합니다.

---

#### **3.7. 플레이어 이탈 처리**
* **FR-701:** 플레이어가 게임 도중 접속을 종료하면, 게임은 중단되지 않고 남은 플레이어들끼리 계속 진행됩니다.
* **FR-702:** 플레이어 이탈 시, 모든 참가자의 채팅창에 **"Player X has left the Game. Still, Player X might be human or AI"** 라는 시스템 메시지가 표시됩니다.
* **FR-703:** 이탈한 플레이어는 투표 대상에서 제외되지 않으며, 최종 결과 화면에 AI였는지 사람이었는지 함께 공개됩니다.

---

#### **3.8. AI 플레이어 규칙 및 로직**
* **FR-801 (AI 모델):** AI의 모든 대화 및 판단은 **OpenAI의 GPT-4 모델 API**를 통해 이루어집니다.

* **FR-802 (페르소나 시스템):**
    * **FR-802-1:** 각 게임이 시작될 때, AI는 사전에 데이터베이스에 정의된 **10개의 페르소나 중 하나를 무작위로 배정**받습니다.
    * **FR-802-2:** 각 페르소나는 다음 속성을 가집니다: **오타 확률, 인터넷 밈 사용 확률, 평균 응답 시간, 평균 답변 길이.** AI는 이 페르소나에 맞춰 응답 스타일을 조절합니다.

* **FR-803 (턴 별 행동 결정):**
    * **FR-803-1:** AI는 자신의 채팅 턴마다 두 가지 API 호출을 순차적으로 수행합니다.
    * **FR-803-2 (1단계 - 투표 결정):** 현재까지의 대화 내용을 API에 전달하며 아래와 같이 질문합니다.
        > "당신은 상대방(들)이 인공지능인지 사람인지 판단해야 합니다. 인공지능일 가능성이 50%를 넘어간다고 판단되면 투표할 수 있습니다. 이번 턴에 투표하시겠습니까? 투표하려면 'Vote', 아니면 'Skip'만 응답하세요."
    * **FR-803-3:** API 응답이 "Vote"이면, 해당 AI는 'Vote' 버튼을 누른 것으로 처리되어 상단 UI("n/N Selected Vote")에 반영됩니다.
    * **FR-803-4 (2단계 - 채팅 생성):** 페르소나 정보와 대화 내용을 기반으로 실제 채팅 메시지를 생성하여 전송합니다.

* **FR-804 (최종 투표 로직):**
    * **FR-804-1:** 게임 종료 후 투표 시간이 되면, AI는 전체 대화 기록을 API에 전달하여 누구에게 투표할지 결정합니다.
    * **FR-804-2 (1:1 투표):** API는 "Human" 또는 "AI" 중 하나를 반환해야 합니다.
    * **FR-804-3 (1:N 투표):** API는 "Player X, Player Y" 형식으로 AI로 의심되는 플레이어 2명의 라벨을 반환해야 합니다.

* **FR-805 (AI 이탈 로직):**
    * AI 플레이어는 자신의 턴에 **낮은 확률로 게임을 이탈**하는 로직을 가집니다. 이는 사람 플레이어의 이탈과 구별할 수 없도록 하여 게임의 혼란성을 증대시킵니다.

### **4. 비기능 요구사항 (Non-Functional Requirements)**

* **NFR-1 (UI/UX):** 반응형 웹 디자인을 채택하여 데스크톱과 모바일 환경을 모두 지원합니다.
* **NFR-2 (성능):** 채팅은 Supabase 실시간 기능을 통해 지연 없이 전달되어야 합니다.
* **NFR-3 (기술 스택):**
    * **UI 생성:** v0
    * **코드 에디터:** Cursor
    * **백엔드/DB:** Supabase
    * **호스팅/배포:** Vercel
    * **AI 모델:** **OpenAI GPT-4 API**

### **5. 데이터 관리 정책**

* **DMP-1 (데이터 휘발성):** 모든 게임 관련 데이터(대기열, 게임방 정보, 채팅 기록, 투표 기록 등)는 **해당 게임 세션이 완전히 종료되는 시점에 데이터베이스에서 영구적으로 삭제되어야 합니다.**
* **DMP-2 (익명성 보장):** 플레이어를 식별할 수 있는 어떠한 정보(IP 주소 등)도 저장하지 않습니다.

### **6. 데이터베이스 스키마 제안 (Supabase 기준)**

* **`games` table:**
    * `id` (uuid, pk), `game_type`, `status`, `topic_id` (fk), `created_at`
* **`players` table:**
    * `id` (uuid, pk), `game_id` (fk), `player_label`, `player_color`, `is_ai`, `persona_id` (fk, nullable), `status` (enum: 'active', 'left')
* **`messages` table:**
    * `id` (bigint, pk), `game_id` (fk), `player_id` (fk), `content`, `created_at`
* **`votes` table:**
    * `id` (bigint, pk), `game_id` (fk), `voter_player_id` (fk), `voted_for_player_id` (fk)
* **`topics` table (신규):**
    * `id` (int, pk), `content` (text) - 100개의 주제 사전 저장
* **`ai_personas` table (신규):**
    * `id` (int, pk), `name`, `description`, `typo_chance` (float), `meme_chance` (float), `avg_response_time_ms` (int), `avg_response_length` (int)# 깃 업로드 테스트
